# Cloud Computing – Grundlagen

## Definition

Nach **NIST** ist Cloud Computing ein Modell, das bedarfsgerechten Netzwerkzugriff auf einen geteilten Pool konfigurierbarer IT-Ressourcen (Netzwerke, Server, Speicher, Anwendungen, Dienste) ermöglicht, die schnell und mit minimalem Verwaltungsaufwand bereitgestellt werden können.

## 5 Merkmale einer Cloud (NIST)

1. **On-Demand Self-Service** – Ressourcen werden selbständig und automatisiert bezogen.
2. **Broad Network Access** – Zugriff über das Netzwerk mit Standardgeräten.
3. **Resource Pooling** – Gemeinsame Nutzung von Ressourcen (Multi-Tenant).
4. **Rapid Elasticity** – Schnelle, oft automatische Skalierung nach oben und unten.
5. **Measured Service** – Messbare Nutzung und transparente Abrechnung (Pay-per-Use).

## Cloud Deployment-Modelle

- **Public Cloud** – Öffentlich über das Internet bereitgestellt (z. B. AWS, Azure).
- **Private Cloud** – Für eine einzelne Organisation, intern oder extern betrieben.
- **Hybrid Cloud** – Kombination aus Private und Public Cloud.
- **Community Cloud** – Geteilt von Organisationen mit gemeinsamen Anforderungen.
- **Multi-Cloud** – Gleichzeitige Nutzung mehrerer Anbieter.

## Cloud Service-Modelle

| Modell                | Beschreibung                           | Beispiel                             |
| --------------------- | -------------------------------------- | ------------------------------------ |
| **IaaS**              | Infrastruktur (VMs, Storage, Netzwerk) | AWS EC2, Azure VM                    |
| **PaaS**              | Plattform/Laufzeitumgebung             | Azure App Service, Google App Engine |
| **SaaS**              | Fertige Anwendungen                    | Microsoft 365, Salesforce            |
| **FaaS / Serverless** | Funktionen on demand                   | AWS Lambda                           |
| **CaaS**              | Container-Plattform                    | AKS, GKE, EKS                        |
| **DBaaS**             | Datenbanken als Service                | Amazon RDS, Cosmos DB                |

## Cloud-Dienstleistungen

Typische Kategorien: Compute, Storage, Datenbanken, Networking, Security & Identity (IAM), AI/ML, Analytics & Big Data, DevOps-Tools, IoT, Monitoring & Logging.

## Cloud-Anbieter

- **Hyperscaler:** AWS, Microsoft Azure, Google Cloud Platform (GCP)
- **Weitere international:** IBM Cloud, Oracle Cloud, Alibaba Cloud
- **Europa / Schweiz:** Swisscom Cloud, Exoscale, OVHcloud, Hetzner, IONOS
- **SaaS-Anbieter:** Salesforce, ServiceNow, Microsoft 365

## Monitoring vs. Logging

| Aspekt        | Monitoring              | Logging                       |
| ------------- | ----------------------- | ----------------------------- |
| **Datentyp**  | Numerische Metriken     | Textbasierte Ereignisse       |
| **Frage**     | „Wie geht's jetzt?"     | „Was ist passiert?"           |
| **Zeitbezug** | Echtzeit / Gegenwart    | Chronologisch / Vergangenheit |
| **Zweck**     | Alarmierung, Dashboards | Fehlersuche, Audit, Forensik  |

In der Praxis ergänzen sie sich: Ein Alarm aus dem Monitoring führt zur Analyse in den Logs.

## Vorteile der Cloud

- Skalierbarkeit und Elastizität
- Pay-per-Use (OpEx statt CapEx)
- Schnelle Bereitstellung (Time-to-Market)
- Hohe Verfügbarkeit und Redundanz
- Zugang zu Innovation (KI, Big Data, Serverless)
- Entlastung der internen IT
- Globale Reichweite

## Nachteile der Cloud

- Vendor Lock-in
- Datenschutz & Compliance (DSGVO, revDSG, CLOUD Act)
- Schwierige Kostentransparenz
- Internet-Abhängigkeit
- Datensouveränität
- Komplexität und Know-how-Bedarf
- Risiko durch Fehlkonfigurationen

## Shared Responsibility

Die Sicherheit ist zwischen Anbieter und Kunde aufgeteilt:

- **Anbieter:** Sicherheit _der_ Cloud → Hardware, Rechenzentren, Netzwerk-Backbone, Hypervisor.
- **Kunde:** Sicherheit _in_ der Cloud → Daten, IAM, Konfiguration, Verschlüsselung, Anwendungen.

Die Grenze verschiebt sich je nach Service-Modell:

| Modell   | Verantwortung Kunde                     |
| -------- | --------------------------------------- |
| **IaaS** | hoch (inkl. OS, Middleware)             |
| **PaaS** | mittel (Anwendung, Daten)               |
| **SaaS** | gering (Daten, Benutzer, Konfiguration) |

**Wichtig:** Daten, Benutzerkonten und Berechtigungen bleiben immer in der Verantwortung des Kunden.
