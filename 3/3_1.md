# Container-Technologie

**Was ist Container-Technologie oder Container-Virtualisierung?**
Isolierte Laufzeitumgebungen, die sich den Kernel des Host-Betriebssystems teilen. Anwendung + alle Abhängigkeiten werden in einem Paket gebündelt und laufen überall gleich.

---

**Was sind die Vor- und Nachteile gegenüber VMs?**

- ➕ Sehr schneller Start (Sekunden), wenig Ressourcen, portabel, einfach skalierbar
- ➖ Schwächere Isolation, nur gleicher Kernel möglich, Sicherheitsrisiko bei Kernel-Lücken, Lernkurve

---

**Welche Produkte kennen Sie?**

- VMs: VMware vSphere/ESXi, Hyper-V, VirtualBox, KVM, Proxmox
- Container: Docker, Podman, Kubernetes, OpenShift, containerd, LXC

---

**Wie unterscheiden sich VM und Container?**

- Bereitstellung: VM in Minuten, Container in Sekunden
- Speicherplatz: VM mehrere GB, Container nur MB
- Portabilität: Container deutlich höher
- Effizienz: Container effizienter (kein eigenes OS)
- Betriebssystem/Kernel: VM hat eigenen Kernel, Container teilt sich den Host-Kernel

---

**Können VMs immer durch Container ersetzt werden?**
Nein. Nicht bei anderem OS-Kernel (z. B. Windows auf Linux-Host), GUI-Anwendungen, stark stateful Legacy-Software oder wenn vollständige Isolation gefordert ist.

---

**Was bedeuten Container, Image und Registry?**

- Image: unveränderliche Vorlage / Bauplan
- Container: laufende Instanz eines Images
- Registry: zentraler Speicher für Images (z. B. Docker Hub, GitHub Container Registry, Azure ACR)

---

**Weshalb werden laufende Container als unveränderbar (immutable) betrachtet?**
Für Reproduzierbarkeit, einfache Rollbacks und klare Versionierung. Änderungen erfolgen durch ein neues Image, nicht am laufenden Container. Daten werden über Volumes ausgelagert.

---

**Was ist der Unterschied zwischen Self-Managed und Fully Managed?**

- Self-Managed: volle Kontrolle, eigene Infrastruktur, mehr Aufwand für Updates/Security/Backup, günstiger bei Lizenzen, mehr Know-how nötig
- Fully Managed: Anbieter übernimmt Betrieb (z. B. AWS EKS, Azure AKS, Google GKE), schnell einsatzbereit, weniger Aufwand, höhere Kosten, Vendor-Lock-in
